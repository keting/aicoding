# Research Topics

本页用于整理 AI Coding（Vibe Coding）/ Coding Agent 方向的研究背景、研究对象、核心问题与选题方向，重点服务实验室科研讨论与选题规划。

本实验室聚焦学术与工程结合，尤其关注企业转向 AI Coding 过程中面临的共性技术问题，而非做纯理论研究。

## 一、背景

2025 年以来，以大语言模型（LLM）为核心的 AI Coding 工具从"代码补全"快速演进到"自主完成工程任务"的 Coding Agent 阶段。Andrej Karpathy 提出的 Vibe Coding 概念——开发者用自然语言描述意图、AI 生成代码、人凭运行结果决定是否接受——正在从个人实验走向团队实践。

与此同时，企业软件开发面临深刻转型：传统的需求分析、设计、编码、测试、部署流程正在被 AI 能力重塑，开发者角色从"写代码的人"向"引导 AI 写代码的人"转变。这一转型带来了大量尚未充分研究的技术问题和工程实践问题，也为学术研究提供了丰富的切入点。

## 二、研究对象

- AI Coding 工具与系统（Claude Code、Codex、GitHub Copilot、SWE-agent 等）
- 人与 AI 的协同编程交互过程
- Coding Agent 的任务规划、执行与评估机制
- 企业软件开发流程在 AI 介入下的变化与适应

## 三、研究意义

- **工程层面**：帮助企业理解"AI 能做什么、不能做什么"，制定合理的 AI Coding 落地策略
- **学术层面**：为人机协同、软件工程、智能体系统等方向提供新的研究问题和实证数据
- **实践层面**：为不同技术背景的开发者提供可操作的协作模式参考

## 四、核心研究问题

### 1. 人机分工边界

AI Coding 并非"AI 全自动"或"人全手动"，核心问题是如何划定合理的分工边界。

重点研究方向：

- **任务适配性**：哪些任务适合 AI 主导（如样板代码生成、测试用例编写、文档生成），哪些任务必须人工控制（如架构决策、安全审计、需求确认）？
- **介入时机**：人类应在什么阶段介入——事前（需求约束）、事中（实时审查）还是事后（结果验收）？不同介入策略对代码质量和开发效率的影响如何？
- **能力分层协作**：不同能力层次的开发者（资深工程师 vs 初学者 vs 非技术人员）是否需要不同的人机协作模式？各自的最佳实践是什么？
- **信任与控制**：开发者对 AI 生成代码的信任度如何影响审查行为和最终质量？Vibe Coding 中"不逐行审查"的风险边界在哪里？

### 2. 上下文工程

Coding Agent 的效果高度依赖其获取和利用上下文的能力，这是当前制约 agent 表现的关键瓶颈之一。

重点研究方向：

- **上下文组织与检索**：如何从大规模代码仓库中高效提取与当前任务相关的上下文（文件、函数、依赖关系、历史变更）？
- **上下文窗口策略**：在有限的上下文窗口下，如何决定哪些信息进入 prompt、哪些被压缩或丢弃？
- **跨会话记忆**：如何在多轮交互和多次会话之间保持项目理解的连续性？
- **上下文质量与噪声**：无关或低质量的上下文对 agent 决策的干扰有多大？如何检测和过滤？

### 3. 多 Agent 协作

当单个 agent 的能力边界明确后，多 agent 协作成为提升复杂任务完成度的自然方向。

重点研究方向：

- **任务分解与分配**：如何将复杂的工程任务拆分为多个 agent 可并行或串行执行的子任务？
- **协作模式**：多 agent 之间如何通信、共享上下文、解决冲突（如多个 agent 修改同一文件）？
- **质量保障**：如何设计 review agent、test agent 等角色来保障多 agent 协作产出的质量？
- **人在环中的协调**：在多 agent 系统中，人类的角色是什么——监督者、协调者还是最终决策者？

### 4. 企业转向 AI Coding 模式与方法研究

企业软件开发正在从传统模式向 AI Coding 模式转型，这一过程涉及流程重构、方法论演进和协作模式创新。

重点研究方向：

- **传统流程融合**：现有的需求分析、设计、测试、运维流程如何与 AI Coding 工具融合？code review、CI/CD、版本管理等工程实践在 AI Coding 场景下需要怎样的调整？
- **转型风险管理**：转型过程中的技术债务、安全风险和质量退化如何预防和管理？
- **AI Native 工程方法**：如果从零设计"AI 原生"的开发流程，其核心工程原则是什么（如 prompt-driven development、agent-assisted TDD）？如何建立相应的质量标准和评估体系？
- **研发角色与流程重塑**：产品经理、开发、测试、运维等传统角色在 AI Coding 时代如何重新定义？团队协作工具和工作流如何适配"人 + 多 Agent"的混合团队模式？

## 五、研究平台与实践载体：HALF

HALF（Human-AI Loop Framework）是实验室围绕 AI Coding / Coding Agent 方向建设的基础实验平台，用于支撑人机协同的多 Agent 编程协作、任务分解、Agent 分工、上下文交接、结果追踪和实验数据沉淀。

HALF 不是单纯的工程项目，而是服务于前述研究问题的实验基础设施：

1. **人机分工边界**：通过人工确认、阶段性审查节点、执行结果追踪等机制，研究人在 AI Coding 流程中的介入时机、控制方式与责任边界。
2. **上下文工程**：通过任务上下文组织、handoff prompt、结构化任务记录与历史经验沉淀等机制，研究上下文组织、压缩、传递与复用方法。
3. **多 Agent 协作**：通过 DAG 任务流、process template、dev/test/review/eval agent 角色，研究多 Agent 分工、协作、冲突处理和质量保障机制。
4. **企业 AI Coding 方法论**：通过真实 Git 工作流、issue/branch/PR 协作、agent 可用性管理等机制，探索企业软件开发流程向 AI Native 模式转型的方法。
